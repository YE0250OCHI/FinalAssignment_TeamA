# チームA 最終課題：自動倉庫システム

## 概要

　本プロジェクトでは、以下のプロダクトを開発する

- 制御ノード　：　複数台の自動倉庫を制御・管理するコントローラ
- 実行ノード　：　自動倉庫単体のコントローラ（スタブ）
- 操作ノード　：　ピッキング指示出し端末

### 背景

　A社では、社内工場で部品製造から組み立てまでを一貫して行い、完成品を出荷している。その工場では、各部品を製造ラインから取り出した後、一度エレクターに保管しておき、組み立てライン側で必要になったタイミングで人がピッキングし、指定された組立ラインまで搬送していた。昨今の人手不足問題により、A社でも工場全体での人員配置を見直すこととなり、まずはピッキング要員を減らすべく、自動倉庫システムを導入することとなった。

### システム全容

　自動倉庫は全部で2台、すべて同じ能力を持ったものとし、自動倉庫の入庫側（製造ライン側）と出庫側（組立ライン側）は、コンベアなどの搬送機器によって一体化しているものとする。まず、製造ライン側で、コンベアや搬送機器を介して出庫口は連結しており、オペレーターは1か所で部品を受け取ることが可能

## 主な機能

### 出庫要求の生成・状態管理

- 操作ノード上で、部品の品種を選択し、個数を入力して、出庫要求を出す
- 出庫要求を出してから、実際に出庫が行われ、出庫完了するまでをリアルタイムで確認できる
- 出庫要求後、処理予約中のタスクをキャンセルする

### 在庫管理

- 各品種の個数を、どの自動倉庫で保管しているかを記録
