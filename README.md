# チームA 最終課題：自動倉庫システム

## 概要

### 背景

　A社では、社内工場で部品製造から組み立てまでを一貫して行い、完成品を出荷している。その工場では、各部品を製造ラインから取り出した後、一度エレクターに保管しておき、組み立てライン側で必要になったタイミングで人がピッキングし、指定された組立ラインまで搬送していた。昨今の人手不足問題により、A社でも工場全体での人員配置を見直すこととなり、まずはピッキング要員を減らすべく、自動倉庫システムを導入することとなった。

### システム全容

　本プロジェクトでは、以下のプロダクトを開発する

- 自動倉庫本体（スタブ）　：　自動倉庫本体を表す実行主体ノード（スタブ）
- 端末画面（PWA）　：　ピッキング指示を行う操作ノード
- コントローラ　：　複数台の自動倉庫を制御・管理する制御ノード

　自動倉庫は全部で2台、すべて同じ能力を持ったものとし、自動倉庫の出庫側（組立ライン側）は、コンベアなどの搬送機器によって一体化しているものとする。まず、製造ライン側の搬送担当者は、新しい部品を任意の自動倉庫に保管する。自動倉庫はこれの入庫処理を行い、入庫完了をコントローラにコンベアや搬送機器を介して出庫口は連結しており、オペレーターは1か所で部品を受け取ることが可能

## 主な機能

### 出庫要求の生成・確認

- 操作ノード上で、部品の品種を選択し、個数を入力して、出庫タスクを生成する
- 出庫タスクのライフサイクルの確認（予約中→処理中→取出待ち→完了）
- 出庫要求後、処理予約中のタスクをキャンセルする
- 過去の出庫タスク履歴の閲覧

### 在庫管理

- 各品種の個数を、どの自動倉庫で保管しているかを記録
- 出庫タスクによって、その個数を確保する際に、保管日時の古い順に全ての自動倉庫に対して分割して、出庫指示を出す
- 入庫報告を各自動倉庫から受けて、在庫情報を更新する
