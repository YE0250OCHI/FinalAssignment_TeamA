# ロギング設計書

## 方針

- サーバーの起動・停止を確認できること。
- Razor PagesとAPI通信の応答結果を確認できること。
- JOBの作成から完了までの状態遷移を追跡できること。
- 自動倉庫の装置状態の遷移を追跡できること。
- タイムアウト、業務エラー、システムエラー発生時の調査に利用できること。

## ログレベル定義

|レベル|用途|
|:---|:---|
|INFO|正常な業務イベント、状態遷移、API処理|
|WARN|業務エラー、タイムアウト、未登録端末、状態不整合|
|ERROR|例外、DBエラー、内部処理失敗、通信エラー|

## 出力先

|項目|内容|
|:---|:---|
|出力先|ローカルファイル|
|出力単位|日次更新、日付変更時に新規ファイル化|
|ファイル名|logs/server-yyyyMMdd.log|
|文字コード|UTF-8|

## 出力項目

|項目|内容|
|:---|:---|
|出力日時|ログ出力日時|
|ログレベル|INFO/WARN/ERROR/DEBUG|
|イベント種別|API受信、JOB作成、JOB状態遷移など|
|メッセージ|処理内容|
|JOB番号|対象JOBがある場合に出力|
|商品コード|対象商品がある場合に出力|
|商品ID|対象商品がある場合に出力|
|設備ID|対象設備がある場合に出力|
|IPアドレス|要求元がある場合に出力|
|HTTPメソッド|API要求の場合に出力|
|URI|API要求の場合に出力|
|ステータスコード|API応答の場合に出力|
|エラーコード|業務エラーの場合に出力|
|例外情報|システムエラーの場合に出力|

## 出力対象

### サーバー起動・停止

|イベント|レベル|出力内容|
|:---|:---:|:---|
|サーバー起動|INFO|サーバーが起動したこと|
|サーバー停止|INFO|サーバーが停止したこと|

### API要求・応答

|イベント|レベル|出力内容|
|:---|:---:|:---|
|API要求受信|INFO|HTTPメソッド、URI、要求元IP|
|API正常応答|INFO|HTTPメソッド、URI、ステータスコード|
|API業務エラー応答|WARN|ステータスコード、エラーコード、理由|
|APIシステムエラー応答|ERROR|ステータスコード、例外情報|

### JOB管理

|イベント|レベル|出力内容|
|:---|:---:|:---|
|JOB作成|INFO|JOB番号、JOB種別、商品コード、商品ID|
|JOB割当|INFO|JOB番号、設備ID、商品ID|
|JOB状態遷移|INFO|JOB番号、変更前状態、変更後状態|
|JOBキャンセル|INFO|JOB番号、キャンセル理由|
|JOB完了|INFO|JOB番号、完了日時|
|JOB異常中断|WARN|JOB番号、中断理由|
|JOB異常終了|WARN|JOB番号、異常理由|

### 在庫管理

|イベント|レベル|出力内容|
|:---|:---:|:---|
|在庫予約|INFO|商品ID、商品コード、設備ID|
|在庫予約解除|INFO|商品ID、理由|
|在庫削除|INFO|商品ID、理由|
|在庫登録|INFO|商品ID、商品コード、設備ID|
|在庫不足|WARN|商品コード、要求数、出庫可能数|

### 自動倉庫管理

|イベント|レベル|出力内容|
|:---|:---:|:---|
|オンライン化|INFO|設備ID、IPアドレス|
|オフライン化|WARN|設備ID、理由|
|未登録端末アクセス|WARN|IPアドレス、URI|
|担当外JOB報告|WARN|設備ID、JOB番号、URI|

### JOB配信

|イベント|レベル|出力内容|
|:---|:---:|:---|
|Push配信成功|INFO|JOB番号、設備ID|
|Push配信失敗|WARN|JOB番号、設備ID、理由|
|Pull配信成功|INFO|JOB番号、設備ID|

### タイムアウト監視

|イベント|レベル|出力内容|
|:---|:---:|:---|
|タイムアウト検知|WARN|JOB番号、設備ID、対象状態|
|再割当成功|INFO|JOB番号、変更前設備ID、変更後設備ID|
|再割当待ち|INFO|JOB番号、変更前設備ID|
|再割当不可|WARN|JOB番号、理由|

### エラー報告

|イベント|レベル|出力内容|
|:---|:---:|:---|
|アラーム受信|WARN|設備ID、JOB番号、アラームコード|
|業務エラー発生|WARN|エラーコード、理由|
|システムエラー発生|ERROR|例外種別、例外メッセージ、スタックトレース|

## 出力形式

``` text
{日時} [{レベル}] {イベント種別} {メッセージ} JobId={JOB番号} EquipmentId={設備ID} ItemCode={商品コード} ItemId={商品ID} Status={状態} Ip={IPアドレス} Method={HTTPメソッド} Uri={URI} StatusCode={ステータスコード} ErrorCode={エラーコード}
```

``` text
2026-06-17 20:15:32.123|INFO|JOB_CREATED|出庫JOBを作成しました|JobId=J20260617-01|ItemCode=I01|ItemId=I01-260617-001|
2026-06-17 20:16:10.456|INFO|JOB_STATUS_CHANGED|JOB状態を変更しました|JobId=J20260617-01|From=Assigned|To=Working|EquipmentId=AS01|
2026-06-17 20:17:00.789|WARN|TIMEOUT_DETECTED|JOBがタイムアウトしました|JobId=J20260617-01|EquipmentId=AS01|Status=Assigned|
2026-06-17 20:17:01.012|ERROR|SYSTEM_ERROR|内部エラーが発生しました|Method=POST|Uri=/api/v1/racks/jobs/J20260617-01/initiate|Exception=SqlException
```

## ログ出力しない情報

|項目|理由|
|:---|:---|
|リクエスト本文全体|ログ肥大化防止|
|スタックトレースの通常出力|正常系ログの可読性低下防止|
|Microsoft. System. の自動生成ログ|業務イベントが埋もれるため|

※ 例外発生時は、ERRORログとして例外情報を出力する。

## 実装方針

|項目|内容|
|:---|:---|
|ログライブラリ|NLog|
|出力形式|テキストログ|
|出力単位|日次ファイル|
|出力方法|各サービス処理内で手動出力|
|ログ出力箇所|Controller、Service、BackgroundService|
