# 要件定義

## 業務要件

|項目|内容|
|:---|:---|
|システムの目的|入出庫作業の自動化、在庫管理|
|利用者|出庫側搬送担当者、自動倉庫|
|対象業務|スマホ操作による商品出庫、在庫状況、自動倉庫状況、エラー報告、自動倉庫での入庫作業|
|システム化範囲|スマホ、サーバー、自動倉庫間の入出庫指示・状態管理|
|範囲外|管理画面、商品マスタの登録・編集・削除、会計、発注、配送、自動倉庫の本番アプリなど|
|制限事項|作業中に異常発生した際は商品が管理対象から外れるので、再度入庫するかお客様環境で対応いただく|

## 機能要件

### スマホ側の機能

|機能|内容|
|------|------|
|商品一覧表示|出庫・入庫対象の商品を選択できる|
|出庫依頼|商品ID・数量を指定して出庫JOBを登録する|
|JOB状況確認|登録したJOBの状態を確認する|
|出庫履歴|過去に出庫したJOBの履歴|

### サーバー側の機能

|機能|内容|
|---|---|
|JOB登録|スマホから出庫依頼を受け付ける|
|マクロJOB管理|スマホから登録された出庫依頼（マクロJOB）を管理する|
|マイクロJOB割当|マクロJOBを商品単位のマイクロJOBへ分解して、対象自動倉庫を決定する|
|マイクロJOB管理|マイクロJOBの状態を管理する|
|プッシュ型JOB配信|配信可能な未割当・未配信マイクロJOBを自動倉庫へ通知する|
|プル型JOB配信|自動倉庫からの要求を受けて、対応するマイクロJOBを配信する|
|入庫受付|自動倉庫端末から入庫要求を受け付け、マクロJOBへ登録する|
|商品の入庫登録|入庫作業の状態を管理し、在庫情報へ反映する|
|在庫管理|商品を枝番で個別管理する|
|自動倉庫状態管理|自動倉庫の空き容量や稼働状態を管理する|
|エラー受付|自動倉庫からエラー報告を受け付ける|
|状態通知|必要に応じてスマホへJOB状態を通知する|


### 自動倉庫（スタブ）側の機能

|機能|内容|
|---|---|
|オンライン通知|起動時・復旧時にサーバーへオンライン通知する|
|次JOB取得|サーバーへ次の出庫JOBを問い合わせる|
|入庫作業|自動倉庫端末から入庫要求を生成し、入庫作業を行う|
|JOB開始通知|JOBの処理開始をサーバーへ通知する|
|JOB完了通知|JOBの処理完了をサーバーへ通知する|
|商品取出通知|出庫品が取り出されたことを通知する|
|エラー報告|作業失敗・機器異常をサーバーへ通知する|
|棚使用状況送信|棚の空き状況をサーバーへ送信または応答する|

## データ定義

|項目|説明|詳細データ|
|---|---|---|
|商品|取り扱う商品に関するデータ|商品個別ID、品種番号、商品名、入庫日、出庫日|
|在庫|自動倉庫内の商品保管状態|商品個別ID、品種番号、格納自動倉庫、在庫状態|
|マクロJOB|入出庫依頼単位のJOB|マクロJOB番号、依頼元、入出庫方向、マクロJOB進捗、マクロJOB詳細|
|マクロJOB詳細|マクロJOBの明細（中間データ）|マクロJOB詳細ID、親マクロJOB番号、品種、個数、出庫済み個数、受取済み個数|
|マイクロJOB|自動倉庫に配信される実行単位JOB|マイクロJOB番号、親マクロJOB詳細ID、入出庫方向、進捗状態、対象自動倉庫、対象商品ID|
|自動倉庫|マイクロJOBの実行単位（スタブ）|機器ID、動作状況、実行中入庫JOB、実行中出庫JOB、エラー状態、空き容量|
|エラー状態|システムの異常|発生日時、機器ID、エラー内容|
