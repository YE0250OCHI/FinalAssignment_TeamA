# コンソール設計資料

部品を保管する自動保管棚を模擬したコンソールアプリ。  
サーバーからの出庫指示を受け取って出庫動作を行うほか、コンソール入力によって入庫動作を行う。  
在庫情報はサーバー側のDBに保存するため、入庫時は登録情報をサーバーに送信する。

## 状態

- 起動時
- 待機状態
- 入庫モード
- 出庫モード (POST受信)
- 出庫モード (GETレスポンス受信)

### 起動時
- 起動時サーバーへオンライン化報告
- 在庫情報（商品番号：品種ID-入庫日-枝番）を送信

### 待機状態

- 入庫操作待機 (コンソール入力)
- サーバーからの出庫指示 (POST) を待機
- 5秒毎にサーバーへ出庫指示確認 (GET送信)
  - 入庫・出庫は同時に動作可能
  - 出庫のPOSTとGETはどちらかが動作中はもう片方は無効

### 入庫動作の流れ

- コンソール入力を待機 (品種ID)
- サーバーに入庫情報 (品種ID) を送信
- サーバーからのレスポンス（JOB番号・商品番号）を受信
- エラーレスポンスなし
  - 入庫動作の完了を待機 (今回はコンソール入力で代用)
  - 完了で待機モードに戻る
- エラーレスポンスあり
  - エラー内容を表示
  - 入力待機状態に戻る
- ESC入力で待機状態に戻る

### 出庫動作の流れ

- コンソールに出庫指示の内容を表示
- 出庫動作の完了を待機 (今回はコンソール入力で代用)
- サーバーへ出庫動作の完了を報告 (POST・コンソール入力をトリガーとする)
- 待機状態に戻る

## 表示

アプリ起動
``` text
========== 入出庫棚 ==========
F12:入庫モードへ

==============================
```

出庫動作
``` text
========== 出庫 ==========
F1:出庫操作モード
ESC:非常停止
==============================
JOB番号：XXXX
商品番号：XXX-YYYYMMDD-ZZZZ
出庫完了(Enter)＞
取出完了(Enter)＞

```

入庫動作
``` text
============ 入庫 ============
F2:入庫操作モード
ESC:非常停止
==============================
品種ID＞XXX
JOB番号：XXXX
入庫完了(Enter)＞
```

## ログ

Nlog
- システムログ
- 入出庫ログ

### システムログ
- 通信の内容やエラーを記録する

### 入出庫ログ
- 入庫・出庫の内容を記録
- 動作|品種番号|個数

## 備考
- アプリ起動時にサーバーに対して出庫指示がないかGETで確認
- 入出庫作業中にサーバーからPOSTが来た”処理不可”などのレスポンスを返す
