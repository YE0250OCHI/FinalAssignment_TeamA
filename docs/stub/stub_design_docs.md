# コンソール設計資料

部品を保管する自動保管棚を模擬したコンソールアプリ。  
サーバーからの出庫指示を受け取って出庫動作を行うほか、コンソール入力によって入庫動作を行う。  
在庫情報はサーバー側のDBに保存するため、入庫時は登録情報をサーバーに送信する。

## 状態

- オフライン
- 待機中
- 作業中
- 異常発生中

### オフライン

- 起動直後および異常発生時に移行
- サーバーへオンライン要求を送信
  - 在庫情報（商品番号：品種ID-入庫日-枝番）・空き容量
- オンライン要求のレスポンスでJOB情報を受け取った場合、出庫バッファに格納

### 待機状態

- サーバーからの出庫要求を待機
- 5秒毎にサーバーへ出庫指示問合せ (Pull送信)
- 入庫操作待機 (コンソール入力)
  - 入庫・出庫は同時に動作可能
  - 出庫要求と問合せのレスポンスが同時に来た場合、片方のみ有効
  - 出庫動作中はPull送信を停止

#### 出庫指示の受付

サーバーからのリクエスト

- サーバーからのリクエスト(JOB番号・商品個別ID)を待機
- リクエスト受信時出庫動作中であればエラーレスポンスを送信
- JOB番号を確認し重複があればエラーレスポンスを送信
- 対象の商品個別IDがなければエラーレスポンスを送信
- 出庫バッファに受け取ったJOB情報を格納

出庫指示問合せへのレスポンス

- 出庫動作を行っていない場合5秒毎に問合せを送信
- 200レスポンス(JOB番号・商品個別ID)を受信した場合
  - JOB番号を確認し重複があれば出庫指示を無視
  - 対象の商品個別IDがなければ出庫指示を無視
  - 出庫バッファに受け取ったJOB情報を格納
  - 5秒待機
- 204レスポンスを受信した場合
  - 処理は行わず5秒待機へ

#### 入庫指示の受付

- コンソールから品種番号の入力を待つ
- 棚に空きがあればサーバーに品種番号を送信
  - 空きがなかった場合はエラーメッセージを表示して入力待機に戻る
- サーバーレスポンスでJOB情報 (JOB番号・商品個別ID) を受信
- 入庫バッファに受け取ったJOB情報を格納

### 動作状態

#### 出庫動作

- 出庫JOB開始通知を送信
- コンソールに出庫指示の内容を表示
- 出庫動作の完了を待機 (今回はコンソール入力で代用)
- 出庫JOB完了通知を送信
- ワーク取出の完了を待機 (今回はコンソール入力で代用)
- 商品取出通知を送信
- 待機状態へ移行

#### 入庫動作

- 入庫JOB開始通知を送信
- コンソールに出庫指示の内容を表示
- 入庫動作の完了を待機 (今回はコンソール入力で代用)
- 入庫JOB完了通知を送信
- 待機状態へ移行

### 異常状態

- コンソールからの非常停止入力や通信異常発生時に移行
- 進行中の作業をすべて異常終了し、サーバーへアラーム報告を送信
- コンソール入力でオンライン要求から復帰する

## DB定義

棚在庫テーブル（テーブル名：inventory）
|No.|項目名|カラム名|データ型|長さ|NOT NULL|PK|備考|
|---:|:---|:---|:---|:---:|:---:|:---:|:---|
|1.|棚番号|block_id|int||〇|〇|1～50|
|2.|商品個別ID|item_id|varchar|50|||品種ID-入庫日-枝番|

### DB操作

#### 初期データ作成

在庫テーブルのベースとなる空枠を作成
操作内容:
- [商品個別ID] が空 (Null) のレコードを50個用意
- UPDATE文で商品個別IDを任意の枠に追加

#### 出庫操作
出庫指示のあったデータを削除する
操作内容：
- サーバーから受信した [商品個別ID] と一致するレコードを検索
- 該当レコードの [商品個別ID] を空 (Null) に更新

#### 入庫操作
入庫指示のあったデータを空枠に格納する
操作内容：
- [棚番号] を昇順でスキャン
- [商品個別ID] が空 (Null) のレコードを検索
- 該当レコードの [商品個別ID] を受信したIDに更新

## ログ

Nlog
- システムログ
- 入出庫ログ

### システムログ

- 通信の内容やエラーを記録する

### 入出庫ログ

- 入庫・出庫の内容を記録する
- 動作完了または異常終了でログ更新
- 動作|方向|商品番号

## 備考

- 収納数50個
- ~~アプリ起動時にサーバーに対して出庫指示がないかGETで確認~~
- 入出庫作業中にサーバーからPOSTが来た場合”422：遷移拒否”レスポンスを返す
- コンソール操作で非常停止のエラーが送れるようにする
  - 発生時はオフライン状態に移行し、コンソール操作でオンライン化からやり直す
  - 実行中JOBは異常終了したものとして、ワークを除去する
- 在庫情報の更新タイミング
  - 出庫：作業開始時点で取り出されたものとして在庫情報は削除する
  - 入庫：作業完了まで在庫情報は登録しない
- 通信のイレギュラーは基本異常終了
- Ctrl＋Cへの対策を追加
- JSON変換時に大文字・小文字を無視する
